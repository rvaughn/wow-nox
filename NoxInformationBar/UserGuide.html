Nox Information Bar

User Guide

Version 5.0.0

Description

Nox Information Bar is an add-on for World of Warcraft that lets you display the game or character information important to you - continuously and in real-time. Most other info bars give you a limited selection of data points you can turn on or off, but Nox gives you exactly what you want to see, exactly the way you want to see it.

With the Nox Information Bar, you get the ability to customize both the data displayed on the bar itself, and in the tooltip displayed when you mouse over the bar. Nox and its various bundled extensions give you a huge number of data points and optional formats to choose from, as well as any static text you want to configure. All of this configuration can be done through a convenient GUI panel.

Nox Information Bar comes to you courtesy of ins0mniaque, who created the original addon, and Gruma of Uther, who took over the addon when ins0mniaque became too busy to work on it. We also have to extend thanks to the innumerable addon authors from whom we borrowed code, ideas, and techniques, including Mairelon, Telo, the TitanMod team, and many, many others.

Features

Nox Information Bar has many great features built-in:

  • Slim, clean looks integrate with WoW without using unnecessary screen space.
  • Sixteen fully customizable data slots.
  • A fully customizable 30-line tooltip which by default displays experience stats similar to Telo's Clock, but can be configured to display anything you want.
  • Experience bar overlays showing current and rested experience in a format easier to read than the default experience bar.
  • Automatic resizing to accommodate the configured data.
  • Complete mobility, allowing you to place the bar wherever you like, and optionally lock it in place once there.
  • A GUI panel for customizing the bar, data slots, and tooltip.
  • Compatibility with myAddOns.
  • Clock offsets from server time configurable per-character for local time displays.
  • Many configurable variables for everything from client performance to character honor to combat to inventory and more.
  • Optional format selectors for most variables to display them in various ways.
  • Ability to configure plain text and colors in any display slot.
  • Includes an optional addon to move many default GUI frames where you want them.

What's Included

Nox Information Bar is delivered as several bundled and related addons. Since every addon requires some of WoW's limited addon memory, and since some of these add background processing to the WoW client as things happen in the game world - even if their data isn't being displayed - packaging the addons like this lets you choose exactly which ones you want to have loaded.

Nox Information Bar includes the following addons:

NoxInformationBar This includes the bar itself and many basic variables. This is required for most of the other addons to work.
NoxCombatStats Adds combat-related information to the bar.
NoxEquipmentStats Adds information about equipped items to the bar, including names, types, durability and repair costs.
NoxHonorStats Adds honor-system information to the bar.
NoxMoneyExtension Adds a coin display showing your current money, like the one at the bottom of the backpack.
NoxPlayerStats Adds basic character and realm information to the bar.
NoxSkillStats Adds information about your character's skills to the bar.
NoxMover Allows you to move several UI frames around to make room for the bar - or just because you want to. Can be used without the Nox Information Bar, but can only be configured using the bar's config panel.
NoxSampleExtension Demonstrates how to write extension addons for Nox Information Bar, but adds no useful information by itself.

Installation

To install, simply unzip the NoxInformationBar_vX.X.X.zip file into your <World of Warcraft>\Interface\AddOns directory. If you do not have the "Interface\AddOns" subdirectories in your game directory, create them before unzipping the Nox Information Bar.

If you are already using an older version of the Nox Information Bar, I recommend deleting any old Interface\AddOns\Nox* subdirectories before unzipping the new file. This will prevent conflicts with files and addons that have been removed in the new version.

Using The Bar

Using the Nox Information Bar is easy - it doesn't do much other than show data.

When you first load the bar, it will show up at the top of your screen displaying its default slots. Hovering over the bar with the mouse pointer will bring up the tooltip, which also displays its default configuration at first. Left click on the bar and hold the left mouse button down to drag the bar wherever you like on your screen. Right click on the bar to bring up a brief menu. This gives you the options of resetting the current session variables or of opening the configuration panel. Move the mouse pointer off of the menu for a few seconds or hit the Esc key to close the menu without selecting an option.

The background of the bar tracks your experience in your current level. The purple bar shows your current experience earned, and the blue bar behind it shows how much experience you can earn while staying rested. Once the purple bar passes the blue bar, you no longer earn rested experience until you rest again. Once the purple bar fills the entire background, your character will progress to the next level and the both the blue and purple bars will reset to reflect your new level.

Note that you cannot resize the Nox bar explicitly. Instead you set the size of each of its display slots, and the bar will resize itself to fit them.

Slash Commands

Slash commands let you quickly control the look and behavior of the Nox bar. Note that all of the functionality you can access through slash commands can also be accessed through the right-click menu or the configuration GUI. Conversely, the configuration GUI can do some additional things that the slash commands cannot.

Show the configuration panel:

/noxib config

Show or hide the bar or individual slots:

/noxib show [slot=table]

/noxib hide [slot=table]

/noxib toggle [slot=table]

Lock the bar position:

/noxib lock

/noxib unlock

Customize slot displays:

/noxib resize slot=table width=number

/noxib remap slot=table display='string'

Reset the curent session statistics:

/noxib reset

Configure the tooltip:

/noxib tooltip [state=<'on', 'off'>] [position=<'anchor'>] [x=number] [y=number]

Configure the clock offset from server time:

/noxib clock [offsethour=number] [offsetminute=number]

"table" parameters can be specified as:

numbers"slot=1"
ranges"slot=1-6"
lists"slot=[ 1 3 7 8 ]"

Display strings for the remap command (enclosed in single quotes) define what each slot will display. Variables in each display string (enclosed in braces) will be replaced in the bar with their current values.

Anchor values for the tooltip can be:

'left'
'topleft'
'top'
'topright'
'right'
'bottomright'
'bottom'
'bottomleft'
'center'

Examples

Mark slots 4, 5 and 6 as visible

/noxib show slot=4-6

Display only the clock (in white) in a 100 pixel wide slot

/noxib hide slot=1-9

/noxib show slot=1

/noxib resize slot=1 width=100

/noxib remap slot=1 display='|cffffffff{24hours}:{minutes}'

Display the current framerate in slot 6, followed by a space and then "fps"

/noxib remap slot=6 display='{framerate} fps'

Put the tooltip in the bottom right corner

/noxib tooltip position='bottomright' x=-32 y=32

Shut off the tooltip

/noxib tooltip state='off'

Reset the clock offsets to zero

/noxib clock

Set the clock offset to minus 2 hours and 30 minutes

/noxib clock offsethour=-2 offsetminute=-30

Customizing The Bar

Both the main bar and the tooltip of the Nox Information Bar can be customized to show any information you want. You do this by specifying display strings for the bar to show.

Display Strings

Display strings determine what each element of the Nox bar will show. These are made of up any combination of variables, formats, color codes and plain text.

Variables are special words contained in braces that the Nox bar replaces at runtime with the actual values they represent. Lists of the variables Nox recognizes are shown in the sections that follow. If Nox does not recognize a variable, for instance if it is misspelled or simply not known, it will not replace the variable and it will instead show up exactly as entered on your screen.

Formats are additional words placed inside the braces of a variable that alter the way the variables shows its value. These can for instance be used to choose between integer and decimal number displays, or for other similar changes. These are also listed later, after the variable sections.

Color codes allow you to change the color and transparency of any text entered. These will work for any variables or plain text, except for certain variables that automatically color themselves. These are explained in more detail later.

Plain text is anything else you enter that is not a variable, color code, or format. Nox always displays plain text exactly as you entered it.

For example, in the display string:

Time played: {timeplayed:short}

"Time played: " is plain text, "{timeplayed}" is a variable, and "short" is a format that changes the way the {timeplayed} variable displays.

The Main Bar

The main bar has sixteen display slots that you can turn on and off, resize, and configure independently. Each slot gets its own display string which you specify. You could configure Nox to show one variable in each slot, configure the entire bar to show only one slot with sixteen variables in it, or create any other configuration you choose. The potential combinations are unlimited.

The Tooltip

The tooltip consists of 30 lines, each containing a left half and a right half. A display string entered in the left half of a line will show up left-justified in the tooltip, and a display string in the right half will show up right-justified. If the right half is left blank, then it will be ignored and the left half will fill the full width of the tooltip.

To leave a blank line in the tooltip, put at least one space in the left display string and leave the right half empty. If both halves of a line are empty, the tooltip will ignore the line entirely, and Nox will remove the line the next time you open the configuration GUI. (This is an easy way to move the contents of the tooltip up - just blank the lines you don't want, close the config GUI and open it up again. The blank lines will be gone.)

Tooltip lines cannot be sized. The tooltip will automatically size itself to fit the text it is showing at the time.

Character-specific Overrides

Nox allows each character to override specific groups of options, so that each character can share the common settings, can have their own settings, or any combination of the two. The four option groups that can be customized are the General options, the Clock settings, the Slot contents, and the Tooltip settings and contents.

By default, all characters share one common set of settings. You can give a character private settings by checking the box for the appropriate group (or groups) in the "Character-specific Options" panel in the config GUI. The common settings for that group will be copied over to the character, and you can start editing them immediately.

To switch a character back to the common settings from customized settings, just uncheck the appropriate checkbox in the config GUI. Note that if you make changes to customized settings and turn off the customization checkbox at the same time, the settings changes will not be saved. Only the customization checkbox will be saved in that case. If you want to change both in one edit session, turn off the checkbox first and then press the "Apply" button. Your checkbox change will be saved, and the common settings will be loaded into the config GUI. Now make your changes to the common settings and save them as normal.

Note that the common settings are copied into the character's settings when you customize, but there is not yet a way to copy a character's custom settings to the common settings.

Supported Formats

Formats are a new feature in version 2.0 that let you change the way many of the variables display their values. This one feature adds a whole new layer of configurability to the Nox bar - as well as allowing for whole new types of variables. The sections shown here match up with the allowed formats shown in the tables of variables above.

To use a format, you simply place a colon after the variable name, inside the brackets, and place one of the format names shown below after it. There should be no spaces between the colon and the format name, or between the format name and the end brace. Some formats allow multiple options at once, and in this case you simply separate each option with a comma, again with no extra spaces. Multiple options must be specified in the order shown below.

Except where noted, formats are always optional - if you leave them out, the variable will display itself using a reasonable default format.

For example, to display the total time played in a shortened form, you would use:

{totaltime:short}

To display your current money in a long form, with the numbers and letters colored like coins, you would use:

{money:long,1,1}

To display the total time played in the default format, which happens to be the same as the medium format, you would use:

{totaltime}

Note in this case that since the formats are left out, the colon is left out too.

Number Formats

Numbers can be displayed as integers, or as real numbers with two decimal places.

FormatDescriptionExample
int Displays the value as an integer. Any fractional part is truncated. 10
dec Displays the value as a real number with two decimal places. 21.34

Boolean Formats

Boolean variables express a value that can be true or false. Normally these variables will print either the name "true" or the name "false", depending on their value. Boolean formats let you change either or both of these names.

Boolean formats are expressed as {variable:text if true,text if false}. Either of the text values may be omitted, but note that if you include a false value without a true value, you must still include the comma.

For example, to print "yes" or "no" instead of "true" and "false", you would use:

{factionatwar:Bloodsail Buccaneers,yes,no}

(The "Bloodsail Buccaneers" part is the required "faction" format for this variable, explained below, and is not part of the boolean format. The boolean part follows it.)

If instead you wanted the text "At War" to show up in red only when you were at war, you would use:

{factionatwar:Everlook,|cffff0000At War}

By contrast, this is how you would do it if you only wanted "At Peace" to show up in green:

{factionatwar:Everlook,,|cff00ff00At Peace}

Note the extra comma in the format, reserving the "empty" true value.

See below for a discussion of color codes.

Time Formats

Times can be displayed in forms of different lengths.

FormatDescriptionExample
long Displays the time with each increment spelled out in full. 1 day, 21 hours, 36 minutes, 5 seconds
medium Displays the time with abbreviations for each increment. 1d 21h 36m 5s
short Displays the time in a compacted format, without increment names. 1.21:36:05

Money Formats

Money can be displayed in a great variety of formats affecting not only how it is shown, but even what part is shown. Money values can even be colored like the coins - yellow for gold, grey for silver, and orange for copper. To color the coins, just include a ",1" at the end of the format, or ",1,1" to color the number and the name of the coin, for formats that include names.

Money formats can even be used to extract part of a money value. For instance, if you are only interested in the amount of gold you have, you can use the "gold" format to show only that part. The "as gold" format will show you the same thing, but will also include your silver as a decimal fraction. Note that this behavior is not exactly the same for silver and copper formats. For example "silver" will show you only the silver part, but "as silver" will also include gold in the value as 100's of silver.

This means that a slot configured like:

{money:gold}g {money:silver}s {money:copper}c

will display the same thing as:

{money:short}

Note that the latter takes less processing power from the game, though. (Actually these examples are not strictly the same - the "short" format will omit the gold portion if it is zero, and the silver portion if both gold and silver are zero. The version with three variables will always show all portions, even if they are zero.)

Money variables support the following formats:

as silver[,dec[,1]] Display the total amount as silver, with copper after the decimal. The number format "dec" may optionally be included, but must be included if you want to color the number silver (with ",1"). 102.03

Memory Formats

Memory formats allow you to display memory values as kilobytes or megabytes.

FormatDescriptionExample
kb Display the amount in kilobytes. 21345
mb[,dec] Display the amount in megabytes, to two decimal places. 21.34
mb,int Display the amount in megabytes, truncated to the nearest whole megabyte. 21

Equipment Formats

Equipment "formats" don't actually affect how a variable is displayed, rather, they determine which equipment slot the variable reports about. The equipment format is required for variables that use it. If it is omitted or unknown, the variable that uses it will display "n/a" instead.

The names below can be used for equipment formats. Note that several alias names are included for convenience, but all names work the same way. For example, both "shoulders" and "pauldrons" can be used to refer to the shoulder item, regardless of what is worn there. These are used just like normal format codes. For example, {repaircost:chest} will display the repair cost of the armor worn on your character's chest, and {equipname:trinket1} will return the name of your first equipped trinket.

Skills Formats

Skills "formats" determine which skill a variable applies to, rather than changing the look of the variable. Because of this, skills formats are required for those variables that use them. If the skill format is omitted, misspelled, or if your character does not know the skill named, skills variables will display "n/a" instead.

Normally, skill formats are entered exactly as the names read on your character's skill panel, for example "Leatherworking" or "Daggers". So, for instance, if you wanted to know your current alchemy skill, you could use the variable "{skillrank:Alchemy}". Note the capitalization of the name - it must appear exactly as it does on the skills panel.

Skills variables also support "generic" skill formats. These are special names you can use to refer to your skill when you don't know ahead of time what the real skill name is. The generic names are formed by taking the name of the skill group on the skills panel (for example, "Professions") and adding a number to the end of it which corresponds to the number of the skill in the group. So, for example, "Professions 1" refers to your first profession skill, and "Weapon Skills 5" refers to your fifth weapon skill, as shown on the skills panel.

The generic names for professions ("Professions 1" and "Professions 2") are the most useful, as they let you set up displays that work for all of your characters, regardless of what professions they have. If you use these, you might also find the {skillname} variable useful for returning the real name of the skill. For instance, the variable {skillname:Professions 1} will display the real name of your first profession, no matter what it is.

Faction Formats

Like skill formats, faction "formats" determine which faction a variable applies to, rather than changing the look of the variable. Because of this, faction formats are required for those variables that use them. If the faction format is omitted, misspelled, or if your character has no reputation with the named faction, faction variables will display "n/a" instead.

Faction formats are entered exactly as the names read on your character's reputation panel, for example "Ironforge" or "Cenarion Circle". So, for instance, if you wanted to know your current reputation with Honor Hold, you would use the variable "{reputation:Honor Hold}". Note the capitalization of the name - it must appear exactly as it does on the reputation panel.

Color Codes

Any slot can be colored by entering the following code before it:

|caarrggbb

where

aa is the alpha (transparency) value,

rr is the red value,

gg is the green value, and

bb is the blue value.

All color values are given as two hexadecimal digits from "00" (no color, or transparent in the case of the alpha value) to "ff" (full color, or opaque for the alpha value).

Also, the code:

|r

will restore the color that was in effect before the last color change. Even if you leave off the restore code, your color changes will only affect the slot they appear in - colors will not bleed into adjacent slots. This also means that every slot must be colored independently. Also, each slot can have several color codes, and the restore code is not required between each color change.

For example, a slot containing the display string '|cffff0000{dps}|r dps' will display the current dps in opaque red (ff, ff, 00, 00), and then " dps" in the default color (which is a nearly pure yellow). As another example, the display string '|cffff0000r|cff00ff00g|cff0000ffb' will display a red 'r', a green 'g', and a blue 'b'.

Note that color codes will have no effect when used with variables that color themselves, such as {latency}.
